# Product

Product context, architecture overviews, and feature documentation.

## What Belongs Here

- Product overview documents (auto-seeded by `/hydrate-knowledge`)
- Architecture or technical overviews (high-level, not API specs)
- Feature descriptions and background context
- Known limitations and constraints
- Integration guides or partner documentation

## Tips

- Add docs that help Claude understand *why* the product works the way it does, not just *what* it does
- Reference this folder when asking Claude to write PRDs, analyze tradeoffs, or explain product decisions

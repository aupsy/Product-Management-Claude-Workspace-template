# Knowledge

The living knowledge base for your product. Claude reads this folder to give informed, grounded answers about your product, market, team, and strategy.

## Subfolders

| Folder | What belongs here |
|--------|-----------------|
| `compete/` | Competitive landscape research, competitor profiles, differentiation analysis |
| `metrics/` | Success metrics framework, KPI definitions, dashboard documentation |
| `org/` | Team context, org structure, stakeholder map, RoB processes |
| `product/` | Product specs, architecture overviews, feature context |
| `strategy/` | Roadmap proposals, strategy documents, investment cases |
| `user-research/` | User interviews, personas, usability findings, Jobs-to-be-Done |

## Naming Convention

Strategic documents: `document-name-YYYYMMDD.md`

## Tips

- Add new documents as you create them — even a rough brain-dump is better than nothing
- `/hydrate-knowledge` can seed these folders from a URL or existing docs folder
- Claude uses this context when you ask strategic questions, write PRDs, or prepare for meetings
- The more specific and current the content, the better Claude's responses
